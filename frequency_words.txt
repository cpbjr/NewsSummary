# SYNTAX GUIDE:
# ---------------
# Basic matching          → keyword (case-insensitive partial match)
# Mandatory terms (+)     → +keyword (article MUST contain this)
# Exclusions (!)          → !keyword (filter out articles with this)
# Quantity limits (@)     → @category:N (max N articles per digest)
# Urgent alerts (@urgent) → @urgent:+keyword (triggers Telegram immediately)
# Correlation (@corr)     → @correlation:word1+word2 (both must appear)
#

# NOTES & TIPS
# ============================================================================

# 1. Start Restrictive, Then Loosen
#    Begin with tight filters (+mandatory terms, many !exclusions)
#    Gradually remove restrictions as you see what's missing

# 2. Monitor First Week
#    Review each digest and note:
#    - Missing important stories? (add keywords or remove !exclusions)
#    - Too much noise? (add !exclusions or reduce @category limits)
#    - Wrong articles getting through? (refine +mandatory terms)

# 3. Seasonal Adjustments
#    Update keywords based on current events:
#    - Election years: Add campaign-related terms
#    - Earnings season: Emphasize financial results
#    - Sports playoffs: Boost team-specific terms

# 4. Source Weight + Keywords = Best Results
#    Combine high-weight trusted sources (config.yaml) with
#    well-tuned keyword filters (this file) for optimal curation

# 5. Telegram Alerts Should Be Rare
#    Goal: <5 urgent alerts per day
#    If getting too many, tighten @urgent: conditions

# 6. Test Changes Gradually
#    Modify 2-3 keywords at a time, review results for a few days
#    before making more changes
